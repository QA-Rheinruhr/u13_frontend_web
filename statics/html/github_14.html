GitHub Repository forken

Das Forken eines Repositorys auf GitHub ermöglicht es dir, eine persönliche Kopie des Repositorys unter deinem eigenen GitHub-Account zu erstellen. Dadurch kannst du Änderungen am Repository vornehmen, ohne das Original-Repository zu beeinflussen. Hier ist eine Anleitung zum Forken eines Repositorys:

  1. Rufe die GitHub-Website (https://github.com/) auf und gehe zum Repository, das du forken möchtest.
  2. Klicke auf den Button "Fork" (Forken) in der oberen rechten Ecke der Repository-Seite.
  3. GitHub wird eine Kopie des Repositorys unter deinem GitHub-Account erstellen.
  4. Nachdem das Forken abgeschlossen ist, wirst du zur Seite deines geforkten Repositorys weitergeleitet.
  5. Jetzt kannst du Änderungen an dem geforkten Repository vornehmen.

Beachte, dass das geforkte Repository anfänglich mit dem Original-Repository verbunden bleibt. Das bedeutet, dass wenn das Original-Repository aktualisiert wird, du diese Änderungen übernehmen kannst, um dein geforktes Repository auf dem neuesten Stand zu halten.

Wenn du Änderungen vorgenommen hast und diese zur Verfügung stellen möchtest, kannst du eine Pull-Anfrage (Pull Request) erstellen. Dadurch kannst du dem Eigentümer des Original-Repositorys deine Änderungen vorstellen und darum bitten, dass diese in das Original-Repository übernommen werden.
